Introduction to QGIS

QGIS is a free and open-source option for processing geographic information. Simply search your computer for QGIS to open the program.

Drag and drop any files of interest (e.g. .tifs) into your new project. To calculate vegetation indices, find the color .tif files in the 4_index/indices folders of your pix4d outputs. (default location: Documents/pix4d/YOUR_PROJECT_NAME/4_index/indices…)

Use the layer panel in the bottom left to adjust the order of your layers. If desired, you can uncheck the boxes of any layers you aren’t using to hide them.

  1. If processing toolbox is not displayed, go to Processing → toolbox.

  1. In the processing toolbox, search for “vector grid“ and click “Create grid”

  1. In “Grid type” select “rectangle”
  2. The next box allows you to select the area of interest. Click the three dots on the right, click “select extent on canvas”, and select using the mouse. It is okay for the grid to cover a bigger total area than you will need, you can always trim it later.

  1. In the next two boxes, select your grid size (note: this is in meters… the default values are extremely small and you will need to increase the spacing between the beginning of plots horizontally and vertically. III. Calculating vegetation indices

Vegetation indices, like Excess Green (ExG) and the visible atmospheric resistant index (VARI) are useful indices for determining plant area and health. Healthier plants typically have higher values.

  1. The equation for Excess Green is:
    2 * (    Green / (Red + Green + Blue)) – (Red / (Red + Green + Blue)) – (Blue / (Red + Green + Blue))

    The equation for VARI is:
    (    Green - Red) / (Green + Red - Blue)
    For this exercise, we will continue using VARI.
  2. Go to Raster → Raster calculator.

 

  1. In the raster calculator, we can create equations and generate maps for vegetation indices. Start by clicking the three dots to the right of “Output layer” to give your file a name and folder when saved.
  2. In the box at the bottom (“Raster calculator expression”), write your mathematical expression using either your keyboard or the operators provided in the window.
  3. To select wavelengths of interest, double-click the corresponding raster band at the top left of the window.
  4. For RGB orthomosaics:

Red    =   IMAGE_NAME@1

Green  =   IMAGE_NAME@2

Blue    =   IMAGE_NAME@3

V. Distinguishing between plant canopy and soil

Want to calculate the canopy area of your plots? To do this, we will need to come up with a threshold for VARI. Above this value, we will consider everything to be living plant canopy. Soil, dead plant residue, and most other materials will be below it. We will use the Raster Calculator for doing this as well.

  1. Reopen the raster calculator, Raster → Raster Calculator.

  1. As done previously, click the three dots to the right of “output layer” to give a name and directory to the new file you will create.

  1. In the Raster Calculator Expression box, enter an expression as follows:

 (Your vegetation index file name) > threshold value (0.5 works well      for NDVI, use other numbers for other VIs)

  1. The value of the threshold will need to be adjusted for different vegetation indices, and at times for lighting, camera settings, etc.

  1. Check the quality of your threshold by comparing it to your RGB orthomosaic. To do this:
  1. Find the “Layers Panel” at the bottom left.
  2. Uncheck any files other than your RGB orthomosaic and the new threshold file.
  3. Check and uncheck the new threshold file to make sure there is good correspondence between the highlighted area in your threshold file and the original orthomosaic. It should look something like this:

  1. If your threshold file underestimates canopy area, repeat steps 1-5 with a lower value for the threshold. If a lot of soil or other material is counted as canopy, then increase the threshold value.

We will determine canopy area with the zonal statistics tool.

  1. Specify the appropriate raster layer and shapefile layer. Give the outputs a very short (<8 character) prefix. Click the three dots to the right if the “statistics to calculate” and choose what data you want:

  1. Click “OK.”
  2. In the Layers Panel, right click the polygon layer you just used to calculate zonal statistics. Select “Open attribute table”.
  3. Use Ctrl+A and Ctrl+C to copy the whole attribute table.
  4. Open a new Microsoft Excel spreadsheet.
  5. Use Ctrl+V to paste the spreadsheet into your document. The default for the cells is to be numbered the way we read, starting at the top left and moving to the right, then moving down a row and then continuing left to right. Sort your fieldbook by plot row and column, then transfer the data. Check a few plots to confirm that no errors were made.
  6. Close window.

VII. Calculating vegetation indices with soil masked

You can use your shapefiles and VARI map to calculate average VARI values for each plot. The problem with this, however, is that much of the plot area may be soil, water, or other material. In these cases, your average values will largely be based on canopy cover, not canopy health.

How do we determine the health of plants, while ignoring soil or water?

We already made a threshold file that can distinguish between plant and non-plant. Now we will use it to remove non-plant area for each plot. This is called masking.

  1. Open the raster calculator.
  2. Click the three dots at the top right to name your file and give it a folder to save to.
  3. Click the box at the bottom. Then double click the raster bands of interest to plug in the following expression:

“VARI_LAYER_NAME” / “THRESHOLD_LAYER_NAME”

The two files above were described in sections IV and IX, respectively. The file window should look like this:

  1. Click OK.
  2. In a few seconds, a new layer will appear in your layer panel. It will only have color where the values are above your threshold, while soil will be transparent (This is because the non-plants were stored as “0” values in our threshold file, while plants were stored as “1”. Dividing by 0 gives an NA, and the computer ignores the value). Use the zonal statistics plugin to extract plot statistics, as described previously.

VIII. Calculating canopy volume and height

  1. Generate a canopy/soil mask layer (also known as a threshold layer), described in section IX.
  2. Drop the digital surface model (DSM) from your Pix4D outputs into your QGIS project.
  3. Using the raster calculator, input the following equation:


DSM_LAYER / THRESHOLD_LAYER


This will yield the DSM of just the canopies, without the soil. I name this the cDSM.

  1. To determine the height of the soil in the same area, use the following equation:

DSM_LAYER / (THRESHOLD LAYER < 0.5)


I name this soil DSM “sDSM.”

  1. Check that the cDSM and sDSM files formed properly.
  2. Use the zonal statistics plugin to export maximum cDSM values (maximum canopy altitude), mean cDSM values (mean canopy altitude), and median sDSM values (median soil altitude)
  3. Export the results in the attribute table to Excel or R, and in one of these programs subtract the mean cDSM values for each plot from the median sDSM, yielding average canopy height. Subtract maximum cDSM values from median sDSMs to identify maximum height per plot
  4. To determine canopy volume, multiply the mean canopy height times the canopy area from section X.
